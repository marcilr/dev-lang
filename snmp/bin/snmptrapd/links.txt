Understanding Simple Network Management Protocol (SNMP) Traps
In SNMPv2c trap is defined as NOTIFICATION and formatted differently compared
to SNMPv1. It has these parameters:
o sysUpTime - This is the same as Time stamp in SNMPv1 trap.
o snmpTrapOID - Trap identification field. For generic traps, values are
defined in RFC 1907, for vendor specific traps snmpTrapOID is essentially a
concatenation of the SNMPv1 Enterprise parameter and two additional
sub-identifiers, '0', and the SNMPv1 Specific trap code parameter.
o VarBindList - This is a list of variable-bindings.
http://www.cisco.com/c/en/us/support/docs/ip/simple-network-management-protocol-snmp/7244-snmp-trap.html
